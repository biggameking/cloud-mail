# Compatibility entry · product architecture brief

This path remains for project instances and task records created before devrules
2.4.0.

The canonical universal template is
[`product-architecture-brief.md`](product-architecture-brief.md), used with
[`product-architecture-review.md`](../workflows/product-architecture-review.md).
It covers product inputs beyond PRDs and separates:

- applicability from the product-readiness verdict;
- capability role from surface role;
- product/IA facts from visual design and technical architecture facts.

Use the canonical template for new work. Do not maintain a second product model
in this compatibility file. Existing completed briefs remain valid historical
evidence; migrate them only when a new product decision requires review.
